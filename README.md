## Задача

Реализовать алгоритм сложения элементов вектора (редукция):
- Входные данные: Вектор размером от 1 000 до 1 000 000 значений.
- Реализация: 
    - Последовательный вариант на CPU.
    - Параллельный вариант на GPU с применением CUDA.
- Выходные данные: Сумма элементов, время вычисления, расчет ускорения.

---

## Реализация

В работе реализовано два подхода к вычислению суммы:

- **CPU** (`cpu_sum`) — классический последовательный проход по массиву с накоплением суммы в одной переменной. Сложность составляет $O(N)$.
- **GPU** (`reduce_sum_kernel` + `final_reduce_kernel`) — параллельная реализация с использованием **древовидной редукции** (tree-based reduction).
    - **Что распараллелено:** Процесс суммирования разбит на блоки. Каждый поток в блоке суммирует элементы и сохраняет результат в разделяемую память (`shared_memory`). Затем внутри блока происходит каскадное сложение (stride reduction), пока не останется одно значение на блок.
    - **Почему это эффективно:** Вместо последовательного сложения $N$ элементов, мы выполняем операцию за $\log N$ шагов. Использование `shared_memory` позволяет минимизировать обращения к глобальной памяти GPU, что значительно ускоряет процесс.
    - **Особенности:** В коде используется двухуровневая редукция: сначала основные блоки вычисляют частичные суммы, затем финальный кернел собирает итоговый результат.
